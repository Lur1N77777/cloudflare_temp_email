# 初始化/更新 D1 数据库

第一次执行登录 wrangler 命令时，会提示登录, 按提示操作即可

## 初始化数据库

```bash
cd worker
cp wrangler.toml.template wrangler.toml
# 创建 D1 并执行 schema.sql
wrangler d1 create temp-email-db
wrangler d1 execute temp-email-db --file=../db/schema.sql --remote
```

> [!tip] 命名建议
> 数据库名称请使用例如 `temp-email-db`、`cloudflare-temp-email-prod` 这样的名称。

创建完成后，我们在 cloudflare 的控制台可以看到 D1 数据库

![D1](/readme_assets/d1.png)

## 更新数据库 schema

执行 `schema` 更新前，请确认当前部署版本，并查看 [更新日志](https://github.com/dreamhunter2333/cloudflare_temp_email/blob/main/CHANGELOG.md)。

### 1. 先备份远程 D1

任何远程迁移都应先导出数据库；导出失败时不要继续：

```bash
cd worker
mkdir -p ../backups
wrangler d1 export temp-email-db --remote --output=../backups/temp-email-db-before-v1.10.0.sql
```

请将 `temp-email-db` 替换为实际绑定的数据库名称，并确认导出的 SQL 文件非空、已安全保存。

### 2. 执行版本对应的迁移

找到需要执行的 `patch` 文件, 执行, 例如:

```bash
cd worker
wrangler d1 execute temp-email-db --file=../db/2024-01-13-patch.sql --remote
wrangler d1 execute temp-email-db --file=../db/2024-04-03-patch.sql --remote
```

### v1.10.0 的 2026-07-13 迁移

v1.10.0 的认证、并发与 IMAP 变更使用以下 8 个迁移文件：

| 迁移文件 | 用途 | 是否可重复执行 |
| --- | --- | --- |
| `2026-07-13-address-token-version.sql` | 为地址凭证增加 `token_version` | **仅一次**（包含 `ALTER TABLE address ADD COLUMN`） |
| `2026-07-13-auth-rate-limits.sql` | 创建登录退避限流状态 | 可安全重跑 |
| `2026-07-13-inbound-delivery-index.sql` | 增加入站 `delivery_key`、唯一去重索引和地址游标索引 | **仅一次**（包含 `ALTER TABLE raw_mails ADD COLUMN`） |
| `2026-07-13-mail-flags.sql` | 创建持久化 IMAP 系统标记 | 可安全重跑 |
| `2026-07-13-outbound-send-requests.sql` | 创建出站幂等账本 | 可安全重跑 |
| `2026-07-13-registration-challenges.sql` | 创建一次性注册 challenge | 可安全重跑 |
| `2026-07-13-send-mail-quota-state.sql` | 创建日/月发件额度原子状态 | 可安全重跑 |
| `2026-07-13-user-token-version.sql` | 为用户凭证增加 `token_version` | **仅一次**（包含 `ALTER TABLE users ADD COLUMN`） |

全新升级且确认这 8 个文件从未执行过时，可在备份后按以下顺序运行：

```bash
cd worker
wrangler d1 execute temp-email-db --file=../db/2026-07-13-address-token-version.sql --remote
wrangler d1 execute temp-email-db --file=../db/2026-07-13-auth-rate-limits.sql --remote
wrangler d1 execute temp-email-db --file=../db/2026-07-13-inbound-delivery-index.sql --remote
wrangler d1 execute temp-email-db --file=../db/2026-07-13-mail-flags.sql --remote
wrangler d1 execute temp-email-db --file=../db/2026-07-13-outbound-send-requests.sql --remote
wrangler d1 execute temp-email-db --file=../db/2026-07-13-registration-challenges.sql --remote
wrangler d1 execute temp-email-db --file=../db/2026-07-13-send-mail-quota-state.sql --remote
wrangler d1 execute temp-email-db --file=../db/2026-07-13-user-token-version.sql --remote
```

::: danger 不要重复执行三个 ALTER 迁移
`address-token-version`、`inbound-delivery-index`、`user-token-version` 整个文件都只能执行一次；重复执行会因列已存在而失败。其余 5 个迁移使用 `IF NOT EXISTS` 或冲突保护，可安全重跑。若不确定执行历史，请先检查表结构或恢复备份，不要把重复列错误当作成功。
:::
